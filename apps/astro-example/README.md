# Astro Example

An Astro consumer of the leadtype pipeline. It statically renders the shared docs fixture, exposes a `.md` mirror endpoint for agents, and wires up client-side search — all from artifacts generated by `leadtype generate`.

## How it's wired

The source loader reads the monorepo's shared fixture via `createDocsSource` (`src/lib/source.ts`):

```ts
import { createDocsSource } from "leadtype";
import docsConfig from "../../../../examples/shared-docs/docs/docs.config";

export const source = await createDocsSource({
  contentDir: path.join(fixtureRoot, "docs"),
  groups: docsConfig.groups,
  baseUrl: "http://localhost:4321",
});
```

- `src/pages/docs/[...slug].astro` — the docs page. Uses `createGetStaticPaths` and `createLoadPageData` from `leadtype/astro`, renders `page.markdown`, and drives in-page search with `createSearchClient` from `leadtype/search/client`.
- `src/pages/docs/[...slug].md.ts` — the markdown mirror. Uses `createMarkdownStaticPaths` and `createDocsEndpoint` from `leadtype/astro` (exported as `GET`/`HEAD`).
- The `.md` files, `llms.txt`, `llms-full.txt`, search index, and `agent-readability.json` are emitted into `public/` by the build.

## Running it

```sh
bun run --filter astro-example dev
bun run --filter astro-example build
```

Both scripts build the `leadtype` package, run `docs:generate`, then start/build Astro.

## What the build does

`docs:generate` runs `leadtype generate --src ../../examples/shared-docs --out public --base-url http://localhost:4321 --json`. It converts the fixture MDX to markdown under `public/docs/`, writes `public/docs/search-index.json` and `search-content.json`, an `agent-readability.json` manifest, sitemap/robots files, and `public/llms.txt` + `public/llms-full.txt`.

## Relationship to `leadtype init`

To scaffold this wiring into your own app, run `leadtype init --framework astro`. This example differs slightly because it reads the monorepo's shared `examples/shared-docs` fixture instead of a local `docs/` folder.
