# Astro Example

An Astro consumer of the leadtype pipeline. It statically renders the repo's real Leadtype docs, exposes a `.md` mirror endpoint for agents, and wires up client-side search — all from artifacts generated by `leadtype generate`.

## How it's wired

The source loader reads the repo-root `docs/` directory via `createDocsSource` (`src/lib/source.ts`):

```ts
import { createDocsSource } from "leadtype";
import docsConfig from "../../../../docs/docs.config";

export const source = await createDocsSource({
  contentDir: path.join(repoRoot, "docs"),
  nav: docsConfig.navigation,
  baseUrl: "http://localhost:4321",
});
```

- `src/pages/docs/[...slug].astro` — the docs page. Uses `createGetStaticPaths` and `createLoadPageData` from `leadtype/astro`, renders generated markdown as HTML, derives sidebar navigation from `leadtype/navigation`, and drives in-page search with `createSearchClient` from `leadtype/search/client`.
- The `.md` files, `llms.txt`, `llms-full.txt`, search index, and `agent-readability.json` are emitted into `public/` by the build. Server or hybrid Astro apps can use `createDocsEndpoint` from `leadtype/astro`; this static example serves the generated files directly.

## Running it

```sh
bun run --filter astro-example dev
bun run --filter astro-example build
```

Both scripts build the `leadtype` package, run `docs:generate`, then start/build Astro.

## What the build does

`docs:generate` runs `leadtype generate --src ../.. --docs-dir docs --out public --base-url http://localhost:4321 --json`. It converts the Leadtype docs MDX to markdown under `public/docs/`, writes `public/docs/search-index.json` and `search-content.json`, an `agent-readability.json` manifest, sitemap/robots files, and `public/llms.txt` + `public/llms-full.txt`.

## Relationship to `leadtype init`

To scaffold this wiring into your own app, run `leadtype init --framework astro`. This example differs slightly because it reads the monorepo's root docs and navigation config instead of a local `docs/` folder.
