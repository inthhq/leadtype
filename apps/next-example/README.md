# Next.js Example

A Next.js App Router consumer of the leadtype pipeline. It renders the repo-root Leadtype docs as MDX pages, serves `.md` mirrors plus `llms.txt` for agents, and proxies agent-readability headers — all from artifacts generated by `leadtype generate`.

## How it's wired

The app uses two build-time inputs:

- `leadtype generate --src ../.. --docs-dir docs --out public --base-url http://localhost:3000 --json` emits `.md` mirrors, `llms.txt`, `llms-full.txt`, search JSON, sitemap/robots files, and `public/docs/agent-readability.json`.
- `scripts/build-mdx-map.mjs` scans the repo-root `/docs` MDX files and writes `app/generated/docs-mdx-map.ts`, which lets the App Router import and render the same source MDX at build time.
- `app/docs/[[...slug]]/page.tsx` derives static params from that MDX map, reads navigation and metadata from the generated agent-readability manifest, renders MDX with the same Leadtype component names as the TanStack app, and emits JSON-LD from `leadtype/llm/readability`.
- `proxy.ts` uses `createDocsProxy` from `leadtype/next` for agent/markdown responses, sitemap routes, and robots routes without shadowing normal HTML pages.

## Running it

```sh
bun run --filter next-example dev
bun run --filter next-example build
```

Both scripts build the `leadtype` package and prepare the docs artifacts. Local server scripts use `portless` for HTTPS on :443.

## What the build does

`docs:prepare` runs `docs:generate`, then writes the MDX import map. That keeps the human HTML route and generated agent artifacts pointed at the same root docs corpus.

## Why `--webpack`?

The `dev`/`build` scripts pin `next dev|build --webpack` instead of the Next 16 Turbopack default. `@next/mdx` passes leadtype's remark plugins (from `createMdxSourcePlugins()`) to its MDX loader as function-valued options, and Turbopack requires loader options to be JSON-serializable — so a Turbopack build fails with *"loader … does not have serializable options."* Webpack has no such restriction. Drop the flag once `@next/mdx` supports non-serializable Turbopack loader options.

## Relationship to `leadtype init`

To scaffold this wiring into your own app, run `leadtype init --framework next`. This example differs slightly because it reads the monorepo's root docs and generated artifact manifest instead of a local `docs/` folder.
