# Next.js Example

A Next.js App Router consumer of the leadtype pipeline. It renders the shared docs fixture as MDX pages, serves a `.md` mirror plus `llms.txt` for agents, and proxies agent-readability headers — all from artifacts generated by `leadtype generate`.

## How it's wired

The source loader reads the monorepo's shared fixture via `createDocsSource` (`lib/source.ts`):

```ts
import { createDocsSource } from "leadtype";
import docsConfig from "../../../examples/shared-docs/docs/docs.config";

export const source = await createDocsSource({
  contentDir: path.join(fixtureRoot, "docs"),
  groups: docsConfig.groups,
  baseUrl: "http://localhost:3000",
});
```

- `app/docs/[[...slug]]/page.tsx` — the docs route. Uses `createGenerateStaticParams`, `createGenerateMetadata`, and `createLoadPageData` from `leadtype/next`, then renders `page.markdown` with `MDXRemote`. JSON-LD comes from `createDocsJsonLd` (`leadtype/llm/readability`).
- `proxy.ts` — `createDocsProxy` from `leadtype/next` serves agent-readability headers for `/docs/*`, `sitemap.xml`, `sitemap.md`, and `robots.txt`.
- The `.md` mirror, `llms.txt`, `llms-full.txt`, search index, and `agent-readability.json` are emitted into `public/` by the build.

## Running it

```sh
bun run --filter next-example dev
bun run --filter next-example build
```

Both scripts build the `leadtype` package, run `docs:generate`, then start/build Next.

## What the build does

`docs:generate` runs `leadtype generate --src ../../examples/shared-docs --out public --base-url http://localhost:3000 --json`. It converts the fixture MDX to markdown under `public/docs/`, writes `public/docs/search-index.json` and `search-content.json`, an `agent-readability.json` manifest, sitemap/robots files, and `public/llms.txt` + `public/llms-full.txt`.

## Why `--webpack`?

The `dev`/`build` scripts pin `next dev|build --webpack` instead of the Next 16 Turbopack default. `@next/mdx` passes leadtype's remark plugins (from `createMdxSourcePlugins()`) to its MDX loader as function-valued options, and Turbopack requires loader options to be JSON-serializable — so a Turbopack build fails with *"loader … does not have serializable options."* Webpack has no such restriction. Drop the flag once `@next/mdx` supports non-serializable Turbopack loader options.

## Relationship to `leadtype init`

To scaffold this wiring into your own app, run `leadtype init --framework next`. This example differs slightly because it reads the monorepo's shared `examples/shared-docs` fixture instead of a local `docs/` folder.
