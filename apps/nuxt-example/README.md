# Nuxt Example

A Nuxt consumer of the leadtype pipeline. It renders the shared docs fixture through a Nitro API route, serves a `.md` mirror, and adds agent-readability headers via server middleware — all from artifacts generated by `leadtype generate`.

## How it's wired

The source loader reads the monorepo's shared fixture via `createDocsSource` (`lib/source.ts`), lazily memoized behind `getSource()`:

```ts
import { createDocsSource } from "leadtype";
import docsConfig from "../../../examples/shared-docs/docs/docs.config";

sourcePromise ??= createDocsSource({
  contentDir: path.join(fixtureRoot, "docs"),
  groups: docsConfig.groups,
  baseUrl: "http://localhost:3000",
});
```

- `pages/docs/[[slug]].vue` — the docs page. Fetches page data from `/api/docs` and drives search with `useLeadtypeSearch` from `leadtype/search/vue`.
- `server/api/docs.get.ts` — resolves page data with `createLoadPageData` from `leadtype/nuxt`.
- `server/routes/docs/[...slug].md.ts` — the markdown mirror, via `createRequiredNitroDocsHandler` from `leadtype/nuxt`.
- `server/middleware/agent-readability.ts` — applies agent-readability behavior with `createNitroDocsHandler` from `leadtype/nuxt`.
- The `.md` files, `llms.txt`, `llms-full.txt`, search index, and `agent-readability.json` are emitted into `public/` by the build.

## Running it

```sh
bun run --filter nuxt-example dev
bun run --filter nuxt-example build
```

Both scripts build the `leadtype` package, run `docs:generate`, then start/build Nuxt.

## What the build does

`docs:generate` runs `leadtype generate --src ../../examples/shared-docs --out public --base-url http://localhost:3000 --json`. It converts the fixture MDX to markdown under `public/docs/`, writes `public/docs/search-index.json` and `search-content.json`, an `agent-readability.json` manifest, sitemap/robots files, and `public/llms.txt` + `public/llms-full.txt`.

## Relationship to `leadtype init`

To scaffold this wiring into your own app, run `leadtype init --framework nuxt`. This example differs slightly because it reads the monorepo's shared `examples/shared-docs` fixture instead of a local `docs/` folder.
