# Nuxt Example

A Nuxt consumer of the leadtype pipeline. It renders the repo's real Leadtype docs through a Nitro API route, serves generated `.md` mirrors, and adds agent-readability headers via server middleware — all from artifacts generated by `leadtype generate`.

## How it's wired

The build step reads the repo-root `docs/` directory and writes generated artifacts into `public/`:

```sh
leadtype generate --src ../.. --docs-dir docs --out public --base-url http://localhost:3000 --json
```

- `app/pages/docs/[...slug].vue` — the docs page. Fetches page data from `/api/docs`, derives navigation from `leadtype/navigation`, renders generated markdown through MDC, and drives search with `useLeadtypeSearch` from `leadtype/search/vue`.
- `server/api/docs.get.ts` — resolves page metadata from the generated Agent Readability manifest and reads the generated markdown mirror from `public/docs`.
- `server/middleware/agent-readability.ts` — applies agent-readability behavior with `createNitroDocsHandler` from `leadtype/nuxt`.
- The `.md` files, `llms.txt`, `llms-full.txt`, search index, and `agent-readability.json` are emitted into `public/` by the build.

## Running it

```sh
bun run --filter nuxt-example dev
bun run --filter nuxt-example build
```

Both scripts build the `leadtype` package, run `docs:generate`, then start/build Nuxt.

## What the build does

`docs:generate` runs `leadtype generate --src ../.. --docs-dir docs --out public --base-url http://localhost:3000 --json`. It converts the Leadtype docs MDX to markdown under `public/docs/`, writes `public/docs/search-index.json` and `search-content.json`, an `agent-readability.json` manifest, sitemap/robots files, and `public/llms.txt` + `public/llms-full.txt`.

## Relationship to `leadtype init`

To scaffold this wiring into your own app, run `leadtype init --framework nuxt`. This example differs slightly because it reads the monorepo's root docs and generated artifact manifest instead of a local `docs/` folder.
