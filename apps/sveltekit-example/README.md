# SvelteKit Example

A SvelteKit consumer of the leadtype pipeline. It prerenders the shared docs fixture, serves a `.md` mirror, and wires up client-side search — all from artifacts generated by `leadtype generate`.

## How it's wired

The source loader reads the monorepo's shared fixture via `createDocsSource` (`src/lib/source.ts`), resolving the fixture root from a few candidate paths:

```ts
import { createDocsSource } from "leadtype";
import docsConfig from "../../../../examples/shared-docs/docs/docs.config";

export const source = await createDocsSource({
  contentDir: path.join(fixtureRoot, "docs"),
  groups: docsConfig.groups,
  baseUrl: "http://localhost:5173",
});
```

- `src/routes/docs/[...slug]/+page.server.ts` — the docs loader. Uses `createEntries` and `createLoadPageData` from `leadtype/sveltekit` with `prerender = true`.
- `src/routes/docs/[...slug]/+page.svelte` — renders the page and drives search with `createLeadtypeSearch` from `leadtype/search/svelte`.
- `src/routes/docs/[...slug].md/+server.ts` — the markdown mirror, via `createDocsServerHandler` from `leadtype/sveltekit` (with `publicDir: "static"`).
- The `.md` files, `llms.txt`, `llms-full.txt`, search index, and `agent-readability.json` are emitted into `static/` by the build.

## Running it

```sh
bun run --filter sveltekit-example dev
bun run --filter sveltekit-example build
```

Both scripts build the `leadtype` package, run `docs:generate`, then start/build via Vite.

## What the build does

`docs:generate` runs `leadtype generate --src ../../examples/shared-docs --out static --base-url http://localhost:5173 --json`. It converts the fixture MDX to markdown under `static/docs/`, writes `static/docs/search-index.json` and `search-content.json`, an `agent-readability.json` manifest, sitemap/robots files, and `static/llms.txt` + `static/llms-full.txt`.

## Relationship to `leadtype init`

To scaffold this wiring into your own app, run `leadtype init --framework sveltekit`. This example differs slightly because it reads the monorepo's shared `examples/shared-docs` fixture instead of a local `docs/` folder.
