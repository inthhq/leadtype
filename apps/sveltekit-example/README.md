# SvelteKit Example

A SvelteKit consumer of the leadtype pipeline. It prerenders the repo-root Leadtype docs, serves `.md` mirrors, and wires up client-side search — all from artifacts generated by `leadtype generate`.

## How it's wired

The app reads the generated artifact manifest and markdown files:

- `src/lib/manifest.ts` normalizes `static/docs/agent-readability.json` and derives sidebar, breadcrumbs, and prev/next links through `leadtype/navigation`.
- `src/routes/docs/[...slug]/+page.server.ts` prerenders one page per manifest entry, loads the generated markdown from `static/docs`, and renders it to HTML.
- `src/routes/docs/[...slug]/+page.svelte` renders the docs shell and drives search with `createLeadtypeSearch` from `leadtype/search/svelte`.
- `src/routes/docs/[...slug].md/+server.ts` — the markdown mirror, via `createDocsServerHandler` from `leadtype/sveltekit` (with `publicDir: "static"`).
- The `.md` files, `llms.txt`, `llms-full.txt`, search index, and `agent-readability.json` are emitted into `static/` by the build from the repo-root `/docs` corpus.

## Running it

```sh
bun run --filter sveltekit-example dev
bun run --filter sveltekit-example build
```

Both scripts build the `leadtype` package, run `docs:generate`, then start/build via Vite.

## What the build does

`docs:generate` runs `leadtype generate --src ../.. --docs-dir docs --out static --base-url http://localhost:5173 --json`. It converts the root docs MDX to markdown under `static/docs/`, writes `static/docs/search-index.json` and `search-content.json`, an `agent-readability.json` manifest, sitemap/robots files, and `static/llms.txt` + `static/llms-full.txt`.

## Relationship to `leadtype init`

To scaffold this wiring into your own app, run `leadtype init --framework sveltekit`. This example differs slightly because it reads the monorepo's root docs and generated artifact manifest instead of a local `docs/` folder.
